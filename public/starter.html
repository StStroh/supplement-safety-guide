Welcome to Supplement Safety Bible

This page appears when your application is missing required environment variables. Follow the steps below to configure your deployment and get started.

Required Environment Variables

Your application requires the following environment variables to function:

Client Variables (Vite)

  • VITE_SUPABASE_URL - Your Supabase project URL
  • VITE_SUPABASE_ANON_KEY - Your Supabase anonymous key
  • VITE_STRIPE_PUBLISHABLE_KEY - Your Stripe publishable key

Server Variables (Netlify Functions)

  • SUPABASE_URL - Your Supabase project URL
  • SUPABASE_SERVICE_ROLE_KEY - Your Supabase service role key
  • STRIPE_SECRET_KEY - Your Stripe secret key
  • STRIPE_WEBHOOK_SECRET - Your Stripe webhook signing secret
  • PRICE_ID_PRO_MONTHLY - Stripe price ID for Pro monthly plan
  • PRICE_ID_PRO_ANNUAL - Stripe price ID for Pro annual plan
  • PRICE_ID_PREMIUM_MONTHLY - Stripe price ID for Premium monthly plan
  • PRICE_ID_PREMIUM_ANNUAL - Stripe price ID for Premium annual plan

Setup Instructions

1. Get Supabase Credentials

  1. Go to Supabase Dashboard
  2. Select your project
  3. Go to Settings → API
  4. Copy your Project URL and anon/public key
  5. Copy your service_role key (keep this secret!)

2. Get Stripe Credentials

  1. Go to Stripe Dashboard
  2. Go to Developers → API keys
  3. Copy your Publishable key and Secret key
  4. Create products and prices for Pro and Premium plans
  5. Copy the price IDs for each plan

3. Configure Netlify Environment Variables

  1. Go to Netlify Dashboard
  2. Select your site
  3. Go to Site settings → Environment variables
  4. Click "Add a variable"
  5. Add all required variables listed above
  6. Save your changes

4. Setup Stripe Webhook

  1. In Stripe Dashboard, go to Developers → Webhooks
  2. Click "Add endpoint"
  3. Enter your webhook URL: https://your-site.netlify.app/.netlify/functions/stripe-webhook
  4. Select these events:
    • checkout.session.completed
    • customer.subscription.updated
    • customer.subscription.deleted
    • invoice.payment_succeeded
    • invoice.payment_failed
  5. Copy the webhook signing secret
  6. Add it to Netlify as STRIPE_WEBHOOK_SECRET

5. Redeploy Your Site

  1. In Netlify Dashboard, go to Deploys
  2. Click "Trigger deploy" → "Clear cache and deploy site"
  3. Wait for the deployment to complete
  4. Your site should now load without the configuration error

Plan Tiers

Your application supports three subscription tiers:

  • Starter - Free tier with limited features
  • Pro - Advanced features and higher limits
  • Premium - Unlimited access and priority support

The tier system is automatically managed via Stripe webhooks. When a user subscribes, their plan is synced to the Supabase profiles table.

Troubleshooting

⚠️ Important: Make sure all environment variables are set correctly. Missing or incorrect values will prevent the application from working.

Common Issues

  • Blank screen: Check that all VITE_ prefixed variables are set
  • Authentication errors: Verify Supabase keys are correct
  • Payment failures: Check Stripe keys and webhook configuration
  • Database errors: Ensure migrations have been applied to Supabase

Need Help?

If you continue to experience issues:

  • Check browser console for error messages
  • Review Netlify function logs for server errors
  • Verify all database tables exist in Supabase
  • Ensure RLS policies are properly configured
